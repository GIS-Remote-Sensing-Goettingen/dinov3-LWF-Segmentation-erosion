# Documentation


## Core docs
- `docs/ARCHITECTURE.md`: repository structure, runtime boundaries, and outputs
- `docs/Implementation.md`: detailed workflow walkthrough and major function guide
- `docs/KB.md`: practical knowledge base for running, debugging, and extending the pipeline
- `docs/CHANGELOG.md`: user-visible change history
- `docs/STYLE.MD`: writing conventions for documentation edits

## Reading order
1. Start with `docs/ARCHITECTURE.md` for the package layout.
2. Read `docs/Implementation.md` for the three workflow modes and major runtime functions.
3. Use `docs/KB.md` as the operational reference for runs, outputs, and troubleshooting.
